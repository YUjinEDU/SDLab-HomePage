# 데이터 모델 및 DB 스키마 초안

상위 문서: [[홈페이지 계획]]

이 문서의 역할:

- 데이터 구조와 DB 테이블 초안을 정의하는 문서

## 공통 규칙

공통 컬럼:

- `id`
- `slug`
- `created_at`
- `updated_at`
- `published_at`
- `visibility`
  - `public`, `member`, `professor`, `admin`
- `status`
  - `draft`, `published`, `archived`
- `display_order`

운영 원칙:

- 공개 페이지 노출 데이터는 `status = published`
- 내부 페이지는 `visibility`와 사용자 역할을 함께 검사
- 홈 화면 Featured 영역은 `is_featured` 또는 별도 연결 테이블로 관리

## 핵심 테이블

### users

- 로그인 계정
- 역할과 계정 상태를 함께 관리

권장 필드:

- `id`
- `email`
- `name`
- `image`
- `role`
  - `member`, `professor`, `admin`
- `account_status`
  - `invited`, `active`, `inactive`, `suspended`, `archived`
- `member_id`
  - `members.id`와 선택적 연결
- `must_change_password`
- `is_active`
- `last_login_at`
- `invited_at`
- `activated_at`
- `created_at`
- `updated_at`

운영 원칙:

- `role`은 권한
- `account_status`는 계정 사용 상태
- 두 값은 분리해서 관리
- 관리자 발급 계정은 기본적으로 `account_status = invited`
- 첫 로그인 후 비밀번호 변경이 끝나면 `active`로 전환 가능

상태별 처리:

- `invited`
  - 첫 로그인 가능
  - 일반 내부 페이지 진입 전 비밀번호 변경 유도
- `active`
  - 정상 사용 가능
- `inactive`
  - 운영상 일시 비활성
- `suspended`
  - 로그인 차단 또는 강한 접근 제한
- `archived`
  - 졸업 또는 사용 종료 후 보관

### members

- 연구실 구성원 프로필
- 이름, 과정, 연락처, 링크, 키워드, 소개, 이미지, 정렬 정보 포함

하위 연결:

- `member_education`
- `member_career`

### research_areas

- 연구 분야 분류
- 제목, 설명, 이미지, 아이콘, 정렬 정보 포함

### publications

- 논문, 저널, 학회, 특허, 보고서 등 연구성과
- 타입, venue, 날짜, DOI, PDF, BibTeX, 키워드 포함

조인:

- `publication_authors`
- `publication_research_areas`
- `project_publications`

### projects

- 연구 과제, 내부 프로젝트, 데모 연계 프로젝트
- 상태, 기관, 기간, 예산, 썸네일, 태그 포함

조인:

- `project_members`
- `project_publications`
- `project_research_areas`

### demos

- Live Demo 콘텐츠
- 카테고리, 썸네일, 데모 링크, 문서 링크, 공개 범위 포함

### board_posts

- 공지, 수상, 행사, 논문 채택, 모집 글 통합 모델
- 카테고리, 요약, 본문, 행사 정보, 고정 여부 포함

### calendar_events

- 내부 일정
- 이벤트 타입, 시간, 장소, 권한 포함

### resources

- 내부 자료실
- 문서, 가이드, 링크, 슬라이드, 저장소 링크 포함

### professor_tools_runs

- 교수님 전용 자동화 실행 이력
- 입력값, 결과값, 실행 상태, 실행 시간 포함

## 관계 원칙

- 멤버와 논문: 다대다
- 멤버와 프로젝트: 다대다
- 프로젝트와 논문: 다대다
- 연구 분야는 프로젝트 / 논문 / 데모와 연결
- 게시글은 프로젝트 / 논문과 선택적으로 연결
- `users.member_id -> members.id`는 선택적 1:1 연결

## 인증 / 계정 관리 관련 규칙

관련 문서:

- [[07 인증 및 계정관리 설계]]

원칙:

- 공개 회원가입은 두지 않는다
- 계정은 관리자 발급 방식으로 운영한다
- 모든 `member`가 `user`를 가지는 것은 아니다
- `admin`은 `member` 연결 없이 존재할 수 있다

초기 계정 생성 흐름:

1. 관리자 계정 생성
2. `role` 지정
3. 필요 시 `member_id` 연결
4. `account_status = invited`
5. 첫 로그인 후 `must_change_password = true`
6. 비밀번호 변경 완료 후 `account_status = active`

## 페이지별 데이터 조회 기준

### 홈

- featured research areas
- featured publications
- featured projects
- latest board posts
- featured members

### Members

- 활성 멤버 전체
- 그룹별 정렬
- 대표 멤버 상세

### Publications

- publication 목록
- 타입 / 연도 / 연구분야 필터

### Projects

- project 목록
- 카테고리 / 상태 / 연구분야 필터

### Board

- 게시글 목록
- 카테고리별 개수
- 검색 대상 필드

### Live Demo

- 공개 가능 데모 목록
- 카테고리 목록
- 로그인 상태에 따른 추가 노출

### Internal

- 내부 공지
- 일정
- GPU 상태
- 내부 자료

### Account / Auth

- 현재 로그인 사용자
- 연결된 member 정보
- 역할
- account status
- 첫 로그인 여부

조회 원칙:

- `account_status = active`가 아니면 보호 라우트 접근 제한 가능
- `invited` 상태는 첫 로그인 플로우 우선
- `suspended`, `archived`는 일반 내부 진입 차단
