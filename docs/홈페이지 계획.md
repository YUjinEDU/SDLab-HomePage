# 연구실 홈페이지 구축 계획

이 프로젝트는 `연구실 소개 홈페이지 + 내부 포털`로 설계한다.  
핵심 방향은 `Next.js 기반 커스텀 웹사이트`, `프론트엔드 템플릿 활용`, `관리형 서비스 활용`이다.

## 이 문서의 역할

이 문서는 전체 허브다.  
코딩 에이전트는 먼저 이 문서를 보고, 그다음 `공개 포털 우선 구현` 기준으로 관련 문서를 순서대로 읽으면 된다.

## 빠른 요약

- 공개 사이트와 내부 포털을 하나의 코드베이스에서 관리한다.
- 공개 페이지는 연구실 소개, 멤버, 논문, 프로젝트, 보드, 데모 중심으로 구성한다.
- 내부 기능은 로그인 후 `calendar`, `gpu`, `resources`, `professor tools`로 분리한다.
- 데이터 구조는 `members`, `publications`, `projects`, `board_posts`, `demos`, `users` 중심으로 잡는다.

## 현재 결정 사항

- 프레임워크: `Next.js`
- UI: `Tailwind CSS`, `shadcn/ui`
- 인증: `Auth.js`
- 데이터베이스: `Supabase` 또는 관리형 `Postgres`
- 권한: `public / member / professor / admin`
