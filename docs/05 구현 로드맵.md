# 구현 로드맵

상위 문서: [[홈페이지 계획]]

관련 문서:

- [[06 Next.js 폴더 구조 및 App Router 설계]]

이 문서의 역할:

- 구현 순서와 단계별 우선순위를 짧게 정리한 실행 문서

## 우선순위

1. 공개 홈페이지 시안 완성
2. 로그인 및 내부 페이지 보호
3. 내부 포털 핵심 기능 연결
4. 교수님 전용 기능 추가

## 단계별 계획

### 1단계. 프로젝트 초기 세팅

- Next.js 프로젝트 생성
- Tailwind CSS 설정
- shadcn/ui 설치
- 기본 레이아웃 구성
- 헤더 / 푸터 / 네비게이션 설계

### 2단계. 공개 페이지 먼저 완성

- 홈
- Members
- Research
- Publications 목록
- Projects 목록
- Board 목록
- Contact

목표:

- 교수님께 보여줄 수 있는 시안 확보

### 3단계. 상세 페이지 및 데모 영역

- Publications 상세
- Projects 상세
- Board 상세
- Live Demo 목록 / 상세

### 4단계. 인증 및 권한

- 로그인 페이지 구현
- Auth.js 연결
- 사용자 역할 정의
- 로그인 상태에 따른 메뉴 분기
- 보호된 라우트 구성

### 5단계. 내부 포털 구축

- `/internal`
- `/internal/calendar`
- `/internal/gpu`
- `/internal/resources`

### 6단계. 교수님 전용 기능

- `/professor`
- `/professor/lecture`
- 실행 이력 저장

### 7단계. 운영 구조 강화

- 관리자용 콘텐츠 입력 구조
- CMS 또는 어드민 대시보드 검토
- 검색 / 통계 / 자동화 기능 확장

## 구현 원칙

- 공개 페이지와 내부 포털을 같은 코드베이스에서 관리한다.
- 공개 페이지 완성 전 내부 기능에 깊게 들어가지 않는다.
- 데이터 모델이 먼저, UI는 그 위에 쌓는다.
- 임시 예외 처리보다 재사용 가능한 구조를 우선한다.
