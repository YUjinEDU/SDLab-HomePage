## 다음 단계 상세 — Phase 2

Phase 2에서는 인증 시스템과 데이터베이스 전환을 진행한다.

### 2-1. Supabase 설정 및 데이터 마이그레이션

- Supabase 프로젝트 생성 및 환경 변수 설정
- 테이블 스키마 설계 및 마이그레이션 (members, publications, projects, patents 등)
- `data/*.ts` 정적 파일 → `lib/queries/*.ts` Supabase 쿼리 함수로 전환
- Supabase 클라이언트 구성 (client, server, middleware)

### 2-2. 인증 구현

- Supabase Auth 로그인/로그아웃 페이지 구현 (`(auth)/` 라우트 그룹)
- 세션 관리 유틸 (`lib/auth/`)
- 역할(role) 필드 연동 (member, professor, admin)

### 2-3. 라우트 보호

- middleware.ts에서 역할 기반 접근 제어
- `lib/permissions/`에서 서버 사이드 권한 검사
- `components/navigation/RoleAwareMenu` 역할별 메뉴 분기 활성화

### 2-4. 게시판 기능

- `/board` 목록 / 상세 / 작성 페이지
- Server Actions (`actions/board/`) 구현

## 구현 원칙

- 공개 페이지와 내부 포털을 같은 코드베이스에서 관리한다.
- 공개 페이지 완성 전 내부 기능에 깊게 들어가지 않는다.
- 데이터 모델이 먼저, UI는 그 위에 쌓는다.
- 임시 예외 처리보다 재사용 가능한 구조를 우선한다.
