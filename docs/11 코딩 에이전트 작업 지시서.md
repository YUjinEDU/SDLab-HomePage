## 구현 원칙

- `Next.js App Router` 기준으로 구현
- 공개 포털 중심으로 작업
- 그린 브랜딩 기준 적용
- 공통 레이아웃, 카드, 필터 패턴을 먼저 확정
- 페이지 간 연결을 반드시 넣는다

## UI 원칙

- 과장된 스타트업 랜딩 페이지처럼 만들지 않는다
- 밝은 배경, 차분한 그린 포인트, 카드형 정보 구조 유지
- 검색/필터/UI 패턴은 페이지마다 다르게 만들지 않는다
- 빈 상태, 오류 상태, 검색 결과 없음 상태를 포함한다

## 콘텐츠 원칙

- 실제 연구실 정보는 [[10 연구실 기본 정보 정리]]를 우선 사용
- 정보가 없는 섹션은 억지로 채우지 않는다
- 문구는 과장하지 않고 연구실 홈페이지 톤을 유지한다

## 구현 우선순위

1. 공통 레이아웃
2. 홈
3. Members
4. Research
5. Publications
6. Projects
7. Contact

## 최소 완료 기준

- 상단 네비게이션과 공통 레이아웃 동작
- 홈에서 연구실 정체성이 바로 보임
- Members / Research / Publications / Projects / Contact 페이지가 연결됨
- 그린 브랜딩이 일관되게 적용됨
- 모바일에서도 레이아웃이 깨지지 않음

## 이번 단계에서 하지 말 것

- 내부 포털 기능 구현에 깊게 들어가지 않는다
- OAuth를 붙이지 않는다
- 관리자 기능을 먼저 만들지 않는다
- CMS부터 붙이지 않는다
- 모든 데이터를 완벽하게 구조화하려고 오래 멈추지 않는다

## 다음 단계
