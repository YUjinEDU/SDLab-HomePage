# 사이트 구조 및 페이지 설계

상위 문서: [[홈페이지 계획]]

이 문서의 역할:

- 공개 포털 중심으로 어떤 페이지를 만들고, 각 페이지를 어떤 컴포넌트로 나눌지 정의하는 화면 설계 문서

- `/research`
- `/publications`

### 인증 영역

- `/login`
- `/account`

## 공통 UX 원칙

- 상단 네비게이션은 전 페이지에서 동일하게 유지한다.
- breadcrumb와 현재 메뉴 강조를 일관되게 적용한다.
- 검색, 필터, 태그, 페이지네이션은 공통 패턴으로 맞춘다.
- 모바일 1열, 태블릿 2열, 데스크톱 3열 기준으로 설계한다.

## 페이지별 컴포넌트 설계

이 섹션은 실제 구현 시 `app/`와 `components/` 구조로 바로 옮길 수 있도록, 페이지를 어떤 컴포넌트 단위로 쪼갤지 정리한 것이다.

기본 원칙:

- 페이지는 `섹션 컴포넌트`로 나눈다.
- 카드, 배지, 필터, 검색창은 가능한 한 공통 컴포넌트로 재사용한다.
- 리스트 페이지와 상세 페이지는 데이터 조회 방식만 다르고, 표현 컴포넌트는 최대한 공유한다.
- 내부 포털과 공개 페이지는 시각 스타일은 다르더라도 작은 UI 단위는 재사용 가능하게 설계한다.

## 공통 재사용 패턴

### 카드 패턴

- `PublicationCard`, `ProjectCard`, `DemoCard`, `MemberCard`는 모두 다음 구조를 공유한다.
  - 상단 메타 영역
  - 제목
  - 요약 또는 핵심 정보
  - 태그 / 상태
  - 하단 액션

### 필터 패턴

- `Projects`, `Publications`, `Board`, `Live Demo`는 동일한 검색 / 필터 / 개수 표시 패턴을 유지한다.
- 상단에는 검색창
- 그 아래에는 필터 또는 카테고리
- 그 아래에는 엔트리 수와 현재 필터 상태

### 상세 페이지 패턴

- 모든 상세 페이지는 기본적으로 다음 구조를 공유한다.
  - `Header`
  - `MetaPanel`
  - `BodySection`
  - `RelatedContentSection`
