  <action>
Migrate `<img>` tags to `next/image` `<Image>` in these files. For each file:

1. Add `import Image from "next/image"` at the top.
2. Replace `<img src=... alt=... className=... />` with `<Image src=... alt=... className=... width=... height=... />`.
3. Remove any `// eslint-disable-next-line @next/next/no-img-element` comments above the img tags.

**Specific guidance per file:**

- **SiteHeader.tsx**: Logo image. Use `width={32} height={32}` (or whatever the current rendered size is). If it is an SVG, set `unoptimized` prop since Next.js cannot optimize SVGs.

- **ProfessorProfile.tsx**: Professor photo. Use `width={200} height={200}` (approximate). Use `className="object-cover"` to maintain aspect ratio. Set `priority` prop since this is above the fold.

- **MemberCard.tsx**: Member avatar thumbnail. Use `width={80} height={80}`. Use `className="object-cover rounded-full"` (preserving existing styles).

- **MembersPageClient.tsx**: Member photos in the grid. Use `width={96} height={96}` (or match existing CSS dimensions).

- **members/[slug]/page.tsx**: Member detail photo. Use `width={200} height={200}`.

- **ProfileForm.tsx**: User's own profile photo preview. Use `width={96} height={96}`.

- **HomeHeroSection.tsx**: If the img is a decorative hero image or logo, use appropriate dimensions. If it is a large hero image, use `fill` prop with `sizes` attribute.

For ALL Supabase storage URLs (member photos), add `unoptimized` prop since the images come from an external domain (unless next.config.ts already has Supabase in `images.remotePatterns` — check first). If remotePatterns is not configured, either add the Supabase domain to `next.config.ts images.remotePatterns` OR use `unoptimized`.

Do NOT migrate canvas-demo or demos page images — those are dev prototypes.
</action>
