    Create frontend/supabase/migrations/004_add_is_public.sql with the following sections in order:

    SECTION 1 — Add is_public columns:
    ```sql
    ALTER TABLE publications ADD COLUMN IF NOT EXISTS is_public BOOLEAN NOT NULL DEFAULT true;
    ALTER TABLE projects     ADD COLUMN IF NOT EXISTS is_public BOOLEAN NOT NULL DEFAULT false;
    ```
    Note: patents is NOT a separate table. publications.type = 'patent'. One column covers both.

    SECTION 2 — Backfill existing rows to match locked decisions:
    ```sql
    UPDATE publications SET is_public = true  WHERE is_public IS DISTINCT FROM true;
    UPDATE projects     SET is_public = false WHERE is_public IS DISTINCT FROM false;
    ```

    SECTION 3 — Replace USING(true) read policies:
    Drop the existing unconditional read policies (confirmed present as USING(true)):
    ```sql
    DROP POLICY IF EXISTS "Public read publications" ON publications;
    DROP POLICY IF EXISTS "Public read projects"     ON projects;
    ```
    Create new policies that allow anon to read only is_public=true rows, but professor/admin see all:
    ```sql
    CREATE POLICY "Public read publications" ON publications
      FOR SELECT USING (is_public = true OR is_admin_or_professor());

    CREATE POLICY "Public read projects" ON projects
      FOR SELECT USING (is_public = true OR is_admin_or_professor());
    ```

    SECTION 4 — update_publication_with_relations RPC (DB-03):
    Create a PostgreSQL function that atomically updates publications + all three join tables.
    The function signature must accept:
      p_id TEXT, p_title TEXT, p_journal TEXT, p_year INT,
      p_volume TEXT, p_issue TEXT, p_pages TEXT, p_doi TEXT,
      p_abstract TEXT, p_type TEXT,
      p_author_ids TEXT[], p_research_area_ids TEXT[], p_project_ids TEXT[]
    Returns VOID. Use LANGUAGE plpgsql SECURITY DEFINER.
    Body:
    1. UPDATE publications SET title=p_title, journal=p_journal, ... updated_at=now() WHERE id=p_id
    2. DELETE FROM publication_authors       WHERE publication_id = p_id
    3. INSERT INTO publication_authors       SELECT p_id, unnest(p_author_ids)
    4. DELETE FROM publication_research_areas WHERE publication_id = p_id
    5. INSERT INTO publication_research_areas SELECT p_id, unnest(p_research_area_ids)
    6. DELETE FROM publication_projects      WHERE publication_id = p_id
    7. INSERT INTO publication_projects      SELECT p_id, unnest(p_project_ids)
    All steps run inside the implicit transaction of a single function call.

    After creating the file, verify the SQL is syntactically valid by checking for obvious errors (balanced BEGIN/END, no missing semicolons).
