  <action>
Create a Node.js ES module script that:

1. **Reads the source MD file** directly via `fs.readFileSync('연구실적-김영국-20251217-2026-03-15_09-47-50.md', 'utf-8')`.

2. **Splits into sections** by finding `<div align="center">` blocks containing Roman numerals II through V. Use regex to identify section boundaries. Skip Section I (Books) and Section VI (Patents).

3. **Extracts HTML tables** from each section. Each section has multiple `<table border="1">` elements (tables split across pages). Use a simple regex or string-based approach to extract table HTML — do NOT use cheerio (avoid adding dependencies). Parse `<tr>` and `<td>` tags with a lightweight custom parser.

4. **Parses each table row** into a publication object:
   - Skip the first row of each table (header row — columns are OCR-corrupted anyway)
   - Skip rows where all cells are empty
   - Map columns positionally: [0]=authors, [1]=title, [2]=venue, [3]=publisher, [4]=date, [5]=volume_info
   - Some rows may have fewer than 6 columns — handle gracefully

5. **Author parsing** (port logic from existing Python parser):
   - Handle "외 N인/명" suffix pattern — extract and append as separate entry
   - Handle comma-separated names (Korean or English)
   - Handle space-separated Korean names where single characters should merge with next token (e.g., "김 영국" -> "김영국")
   - Handle concatenated Korean names without separators (e.g., "박원의강상길김영국") — split into 2-3 character Korean name tokens
   - Handle English names with comma separators
   - Handle mixed formats like "Dongwon Seo 외8명"

6. **Extract index_type** from venue+volume text:
   - Search for patterns: (SCI), (SCIE), (SCOPUS), (KCI) in venue or volume_info text
   - Return the most specific match (SCIE over SCI if both appear)
   - Store as uppercase string or null

7. **Extract DOI** from venue or volume_info text:
   - Regex: `10\.\d{4,}/\S+` then trim trailing punctuation (.,)
   - Also handle `doi.org/10.xxx` format — extract the DOI part after doi.org/

8. **Parse year and month** from date string:
   - Handle formats: "2006.2", "2006.02", "Sept.2013", "2021.01.19.(online)", "22.06.14(online)"
   - Two-digit year prefix: "22." -> 2022
   - Extract first valid year (1993-2026 range)

9. **Clean volume_info**: store the raw Vol/No/pp text from column 5 after whitespace normalization.

10. **Generate output JSON** array where each entry has:

    ```
    {
      title: string,
      authors: string[],
      type: 'journal' | 'conference',
      is_international: boolean,
      venue: string,        // cleaned venue name (without index info)
      publisher: string,    // from column 3
      year: number,
      month: number | null,
      doi: string | null,
      index_type: string | null,
      volume_info: string | null,
      is_featured: boolean, // true if international journal + year >= 2020 + high-impact publisher
      section: string       // 'intl_journal' | 'intl_conference' | 'dom_journal' | 'dom_conference'
    }
    ```

11. **Write output** to `scripts/publications.json` with 2-space indentation.

12. **Print summary** to stdout: count per section and total.

Important implementation notes:

- Use only Node.js built-in modules (fs, path, crypto for uuid). NO npm dependencies.
- Handle the fact that author names in the OCR source are often concatenated without any separator (e.g., "주원군바민우취기석김용김영국"). For these cases, attempt to split into 2-3 character Korean name tokens. This is imperfect — some names will be wrong, but capture what we can.
- Some entries have notes like "(지급 대상 아님)" or "(자체연구비결과물)" in the volume column — preserve in volume_info but don't let them corrupt other fields.
- Filter out entries with title length < 5 characters or no valid year.
- The venue field should have the index info stripped (moved to index_type). Example: "Transactions on Consumer Electronics" not "Transactions on Consumer Electronics (SCI)(IF:2.323)".
