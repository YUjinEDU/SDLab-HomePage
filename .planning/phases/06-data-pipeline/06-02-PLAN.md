  <action>
Create a Node.js ES module script that reads `scripts/publications.json` and inserts data into Supabase.

1. **Load environment variables** from `frontend/.env.local` by reading the file and parsing KEY=VALUE lines (do NOT use dotenv — no npm dependencies). Extract `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`.

2. **Use Supabase REST API directly** via `fetch` (no @supabase/supabase-js dependency). Build requests manually:
   - Base URL: `${SUPABASE_URL}/rest/v1/publications`
   - Headers: `apikey: SERVICE_ROLE_KEY`, `Authorization: Bearer SERVICE_ROLE_KEY`, `Content-Type: application/json`, `Prefer: resolution=merge-duplicates` (for upsert)

3. **Read publications.json** from `scripts/publications.json`.

4. **Generate IDs and slugs** for each entry:
   - `id`: Generate UUID v4 using `crypto.randomUUID()`
   - `slug`: Generate from title — English titles get word-based slug (lowercase, hyphens, max 60 chars); Korean titles get uuid prefix + year. Check for duplicates within the batch and append `-2`, `-3` etc.

5. **Clean and transform** each JSON entry to match DB schema:

   ```
   {
     id: generated uuid,
     slug: generated slug,
     title: entry.title,
     authors: entry.authors,           // TEXT[] — Supabase accepts JSON array for array columns
     type: entry.type,                 // 'journal' or 'conference'
     is_international: entry.is_international,
     venue: entry.venue,
     year: entry.year,
     month: entry.month || null,
     doi: entry.doi || null,
     pdf_url: null,
     abstract: null,
     keywords: [],
     bibtex: null,
     is_featured: entry.is_featured || false,
     is_public: true,
     index_type: entry.index_type || null,
     volume_info: entry.volume_info || null
   }
   ```

6. **Delete existing publications** first (clean slate approach):
   - DELETE from `publication_authors` (junction table)
   - DELETE from `publication_research_areas` (junction table)
   - DELETE from `publication_projects` (junction table)
   - DELETE from `publications`
   - Use Supabase REST API DELETE with no filter (or filter that matches all)

7. **Insert in batches** of 50 rows using POST to Supabase REST API. Log progress (e.g., "Inserted 50/200...").

8. **Auto-link authors to members** after insertion:
   - Fetch all members: GET `/rest/v1/members?select=id,name_ko`
   - For each publication, check if any author name matches a member's name_ko
   - Insert matching rows into `publication_authors` with author_order = position in authors array

9. **Print summary**: total inserted, total author links created, any errors.

10. **Error handling**: If any batch fails, log the error and continue with next batch. At the end, report how many succeeded vs failed.

Important notes:

- Use `fetch` directly (Node.js 18+ built-in). No npm dependencies needed.
- The service role key bypasses RLS, so all operations will work.
- The delete-then-insert approach is safe because this is a one-time seed operation.
- Handle the Supabase array column format: send `authors` as a JSON array, Supabase REST API accepts `["name1", "name2"]` for TEXT[] columns.
